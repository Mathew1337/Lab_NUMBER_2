
# Лабораторная работа №2 по дисциплине "Структуры и алгоритмы обработки данных".

# Система маршрута автобуса

Консольное приложение для управления маршрутом общественного транспорта. Реализовано с использованием структуры данных "Двусвязный связный список".

# Обоснование выбора структуры данных

Для хранения остановок выбран список (динамический массив). Выбор обоснован сопоставлением требований задачи со свойствами линейных структур данных:

Быстрый доступ по индексу и эффективное добавление в конец. В отличие от связного списка, где доступ по индексу требует O(n) времени, массив позволяет получить любую остановку за O(1), что важно для операции поиска остановки через N позиций. Добавление в конец выполняется за амортизированное O(1).

# Функционал

Обязательная часть:

Добавление остановок.

Расчёт общего времени следования по маршруту.

Поиск остановки по порядковому номеру от начала.

Построение обратного маршрута с корректным пересчётом времени перегонов.

Вариативная часть:

Создать функцию формирования подробного отчета маршрута с сохранением в текстовый файл - Задание 4

Сохранение маршрута в текстовый файл, где каждая строка — Base64-кодированная информация об остановке. - Задание 6
